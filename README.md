# 北極星はなぜ動かない？ — 3D 星空 & 太陽系エクスプローラー

地球の自転軸の延長線上にほぼ北極星があるため、地球が回っても北極星だけは
ほとんど動いて見えない — それを 3D で体感できる教育用ビジュアライザです。

## 使い方

`index.html` をブラウザでそのまま開くだけ（ダブルクリックでも OK）。
Three.js は CDN（unpkg）から読み込むのでオンライン環境が必要です。

### 3 つのビュー
- **🛰 宇宙** … 宇宙空間から地球と自転軸・北極星・星座を眺める
- **🗾 日本** … **宮古島（北緯 24.74° / 東経 125.28°）**の地上から夜空を見上げる。
  再生すると星が北極星を軸に回り、地平線の下の星は地面で隠れる（地面トグルで全天表示も可）
- **☀ 太陽系** … 太陽を中心に水金地火木土天海冥が公転（速度比はケプラーの第3法則）

### 忠実な季節・時刻の再現
日付・時刻を指定すると、その瞬間に**宮古島から実際に見える星空**を再現します。
- 恒星時（GMST）で地球の向きを決定 → 季節・時刻ごとに正しい星座が昇る／沈む
- **太陽・月・水星〜海王星**を実際の赤経赤緯（天体暦）で配置。宇宙・日本どちらの視点でも表示
- 「今」ボタンで現在時刻、「時間の速さ」スライダーで早送り

### 機能
- 星にマウスホバーで星名表示（ポルックス、シリウス等）
- 星座検索＆フォーカス（部分一致・「座」省略可）
- 12 星座 / 春夏冬の大三角 / 天の川 / 自転軸 / 地軸の傾き / 地面 などのトグル

### モバイル対応
- スマホ表示でパネルは**☰ ハンバーガー**で開閉、初期は畳まれて画面いっぱいに星空
- **1本指=視点回転 / 2本指ピンチ=ズーム**（OrbitControlsとカスタム実装の両方）
- ボタン・入力欄はタップしやすいサイズに自動拡大
- 操作ヒントもタッチデバイスでは自動的に「指でなぞる / ピンチ」表記に切り替え

## 構成

| ファイル | 役割 |
|----------|------|
| `astro.js` | 純粋な天文・数学ロジック＋星データ（**単一の真実の源**、依存ゼロ） |
| `astro.test.js` | `astro.js` の単体テスト（Node 標準 `node:test`） |
| `index.html` | ビューア。`astro.js` を**インライン展開**して使用（後述） |
| `build.js` | `astro.js` を `index.html` のマーカー間にインライン展開する |

### なぜインライン展開するのか
`index.html` は `file://` で直接開く想定です。ブラウザは `file://` から
ローカル ES モジュールを `import` するのを CORS で禁止するため、`astro.js` を
そのまま `import` できません。そこで `astro.js` を真実の源として保ちつつ、
`build.js` でそのコードを `index.html` の
`// ==ASTRO_INLINE_START==` 〜 `// ==ASTRO_INLINE_END==` の間に
コピー（`export` を除去）して、ダブルクリックで動くようにしています。

## 開発ワークフロー

```bash
# ロジックやデータは astro.js を編集する（index.html のマーカー間は直接編集しない）
node build.js     # astro.js を index.html へインライン展開
node --test       # 52 件の単体テストを実行
```

`npm run build` / `npm test` でも同じことができます。

### テストでカバーしていること
- 座標変換 `raDecToXYZ`（極・赤道・各象限・半径保存）
- 星名解決 `starName`（固有名／バイエル符号／フォールバック）
- 検索 `searchMatch`、観測者の地平線・方位計算、視点エイム
- ケプラー速度比、公転表示、1 年＝約 8.7 秒（約 360 万倍速）の換算
- **天体暦**：ユリウス日・恒星時、太陽の至点／分点での赤緯（±0.4°）、
  月・惑星の赤経赤緯が妥当な範囲に収まること、宮古島の座標
- **星データの整合性**：全リンク端点・labelKey の存在、RA/Dec 範囲、
  12 星座の網羅、惑星の順序、一意性 など
